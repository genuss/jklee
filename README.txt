Результаты храним в таком порядке: base_dir/pid?/profiling_session/{log,result,additional_data}

Запуск профилирования:
* запрос: id, аргументы для async-profiler, формат результата, длительность профилирования.
    Также можно разбить аргументы в будущем: отдельно событие, интервал и т.д.
* ответ: id, код ответа (запущено, уже профилируется, не получилось запустить и т.д.)

Список известных профилирований
* запрос: пусто
* ответ: список из объектов id, статус, ссылки на файлы с логами.
    На будущее можно добавить размер файла с результатами для удобства

Скачивание результатов профилирования
* запрос: id
* ответ: файл

Настройки и эффективные параметры
* запрос: пусто
* ответ: настройки, как они есть: JkleeSettings и эффективные параметры: пути до файлов, путь до и версия async-profiler...
